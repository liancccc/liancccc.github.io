Claude Oplus 4.6 体验

前言

体验一把 anthropic 今年 2 月的新模型 Claude Oplus 4.6。

结论就是除过贵，没啥缺点，至少写代码这块，除了钱欻欻下，几乎是没有什么毛病。

但是后面不会再用了，除非变得很便宜，要不然我还是喜欢古法编程 + AI 简单辅助。

Do

用 claude 去重构一下 threat-sail 项目，因为之前的 WEB 是 fastapi + vue 的 element，然后爬虫的调度和 WEB 是一体的。

爬虫写了一个类，对应的爬虫实现对应方法然后注册一下就执行了。

的确是方便，但是还是想把爬虫写成单独的脚本，不依赖这个框架，后期方便做迁移工作，就是直接放到其他的项目里跑，方便加其他的步骤。

但是真的是懒的修改了，所以也体验一下这个比较厉害的模型。

其实是很震惊的，写出来的东西也不用改，直接就能跑，功能上不用再盯着 AI 指正，大概就 1 个小时左右就自己全部整完了。

给 claude 说了要拆分，然后把需要的模块整理成文档，指出原项目的目录，让它给我重构。让后就欻欻欻的开始。

爬虫 => 去重 => LLM 分析 => WEB

跑了一会，总体来说，功能上没问题。

很丑，因为直接让他参考之前前端的功能，也没具体，不过这只是一轮完成的成果，后面对细节再调整的话是没事问题的。

结语

死贵死贵 不适合日常使用。

本来还以为可以用很久，因为 Deepseek 还有 Kimi 去年还是前年充了 100 块，现在还没用完。

这个直接俩小时干 200 块，还准备体验体验其他的模型，直接凉凉。最初还想着能用个大半年，啧啧。

平常还是 vscode/claude + 国内的模型，或者就 cursor 切账号那种。
